## Why (为什么)

目前代理系统在 `message_request` 和 `usage_ledger` 表中追踪了请求元数据（token 数量、成本、耗时、模型、供应商链路等），但并未捕获用户的实际查询内容。管理员无法看到用户在问什么、如何与编码智能体交互（工具使用模式、对话深度、任务复杂度），也无法识别用户遇到困难的地方。这一缺失导致无法评估用户熟练程度、检测滥用行为，以及基于真实使用模式改进系统。

## What Changes (变更内容)

- 在代理管道中新增查询内容捕获能力，提取并**完整记录**每次请求中用户发送的消息内容（不截断、不摘要），与现有请求元数据关联
- 新增 `query_log` 表，用于存储完整的用户查询内容，支持可配置的保留策略和启用/禁用控制
- 新增使用模式分析功能，将查询数据聚合为行为指标：智能体工具使用频率、对话轮次深度、任务分类、成功/失败模式
- 新增管理员仪表板页面，用于浏览用户查询原文和查看使用分析
- 新增系统设置项，控制查询捕获行为（启用/禁用、保留期限）

## Capabilities (能力)

### New Capabilities (新增能力)
- `query-capture`: 从代理请求中提取并**完整**持久化用户查询内容（原文存储，不截断、不摘要），支持启用/禁用和可配置的保留期限
- `usage-pattern-analytics`: 将捕获的查询数据聚合为行为指标（任务分类、工具使用模式、对话深度、用户熟练度指标），并通过仪表板 UI 展示

### Modified Capabilities (修改的能力)
_(无)_

## Impact (影响范围)

- **数据库**: 新增 `query_log` 表（存储完整查询原文，使用 `text` 类型不设长度限制）；在 `message_request` 上新增查询相关索引；需要数据库迁移
- **代理管道**: 在 `proxy-handler.ts` 中新增守卫或响应后钩子，用于从请求体中提取查询内容
- **数据访问层**: 新增 repository 模块，用于查询存储和分析聚合查询
- **仪表板**: 在 `/dashboard` 下新增页面，用于查询浏览和使用分析图表
- **API**: 新增 Server Actions，用于查询搜索、分析聚合和设置管理
- **国际化**: 新增仪表板 UI 的翻译键（5 种语言）
- **隐私**: 查询内容属于敏感数据；支持可配置保留期限（自动清理）和全局启用/禁用开关
- **性能**: 查询提取必须是非阻塞的（异步写入，类似现有的 `message-write-buffer` 模式）
